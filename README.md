# pre-coding review checklist
before calling `start_coding_task`, verify ALL items:
- [ ] goal is specific and achievable in one coding session
- [ ] each feature has clear acceptance criteria (what user can do, what they see)
- [ ] screen flow documented in docs/screens.md (every screen, navigation, data shown)
- [ ] data model defined (what data is stored, relationships)
- [ ] edge cases addressed (empty states, error states, loading states)
- [ ] UI mockups referenced if user provided any
- [ ] no ambiguous requirements (coding agent cannot ask for clarification)
